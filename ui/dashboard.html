⚙️ Install Agent Into Another Repo
🤖 How It Works After Install
Developer pushes code → PR opens
    ↓
Your CI pipeline runs (tests, build, lint…)
    ↓  ❌ FAILS
cicd-agent.yml triggers automatically
    ↓
Agent downloads log + reads PR diff
    ↓
GPT-4 identifies root cause
    ↓
✅ Commits fix to PR branch → CI re-runs
    OR
📋 Posts step-by-step solution guide on PR
📂 Option A — Copy Files Locally
📋 Option B — Manual Steps (for any remote repo)
1

Copy 3 things into your repo

cp .github/workflows/cicd-agent.yml  YOUR_REPO/.github/workflows/
cp -r agent/   YOUR_REPO/agent/
cp main.py requirements.txt  YOUR_REPO/

Then commit and push:

git add .github/workflows/cicd-agent.yml agent/ main.py requirements.txt
git commit -m "chore: add CI/CD error analysis agent"
git push
2

Add 2 GitHub Secrets

Go to: github.com → YOUR_REPO → Settings → Secrets → Actions

  • OPENAI_API_KEY — your sk-... key from platform.openai.com
  • AGENT_GH_TOKEN — GitHub PAT with repo + workflow scopes

Create PAT at: github.com → Settings → Developer settings → Tokens (classic)

3

Done! 🎉 It's live.

Every future CI failure in that repo will be automatically analyzed and fixed.

Trigger manually anytime by commenting /analyze on any PR.
